  <h1>AKIOS v0.1.0 — Notes de version</h1>
  <div class="post-author">
    <img src="https://github.com/aj14pm.png" alt="Auteur" class="avatar">
    <span>AJ</span>
  </div>
</header>

<div class="post-content">
  <p class="lead">
    Nous sommes ravis d'annoncer la première préversion publique d'AKIOS v0.1.0. Cette version se concentre sur la stabilité du cœur du runtime et les primitives fondamentales de sandboxing.
  </p>

  <h2>Points forts</h2>
  <ul>
    <li><strong>Runtime principal :</strong> première version stable du binaire <code>akios-runtime</code> pour macOS et Linux.</li>
    <li><strong>Moteur de politiques :</strong> définitions de politiques flexibles en YAML pour le réseau et le système de fichiers.</li>
    <li><strong>Outils CLI :</strong> nouveau CLI <code>akios</code> pour gérer le cycle de vie des agents.</li>
  </ul>

  <h2>Architecture en un coup d'œil</h2>
  <p>
    AKIOS sépare plans de contrôle et de données : les politiques sont signées et validées en amont, puis appliquées avant tout effet. Le plan de contrôle garde le dépôt et le moteur de politiques ; le plan de données enchaîne sandbox → garde PII → coupe-circuits budget/boucles → audit. Les agents n’atteignent outils/APIs qu’au travers de ces garde-fous.
  </p>
  <figure class="arch-figure">
    <img src="../assets/img/hero-arch.svg" alt="Clients/CI/Apps passent par le plan de contrôle AKIOS puis le plan de données avant les destinations sous politique">
  </figure>

  <h3>Flux</h3>
  <ul>
    <li><strong>Clients / CI / Apps → Plan de contrôle :</strong> Politiques validées et signées.</li>
    <li><strong>Plan de contrôle → Plan de données :</strong> L’orchestrateur applique les contraintes sandbox.</li>
    <li><strong>Stack sandbox :</strong> seccomp/userns/cgroups → rédaction PII → gardes budget/boucles → audit Merkle.</li>
    <li><strong>Agents → Outils/APIs :</strong> Seuls les appels autorisés passent ; tout est journalisé et attestable.</li>
  </ul>

  <h2>Installation</h2>
  <p>
    Vous pouvez installer la dernière version via Homebrew ou en téléchargeant directement le binaire.
  </p>

  <pre><code>brew tap akios-ai/tap
brew install akios

# Vérifier l'installation
akios --version</code></pre>

  <h2>Changelog</h2>
  <h3>Ajouts</h3>
  <ul>
    <li>Support initial des agents basés sur WASM.</li>
    <li>Ajout du mode <code>--strict</code> pour des paramètres de sécurité très restrictifs par défaut.</li>
    <li>Lancement du site de documentation (que vous êtes en train de consulter).</li>
  </ul>

  <h3>Corrections</h3>
  <ul>
    <li>Correction d'une condition de course dans le logger qui causait des pertes de logs sous forte charge.</li>
    <li>Correction de la gestion des permissions de fichiers pour les montages en lecture seule.</li>
  </ul>

  <h2>Contributeurs</h2>
  <p>
    Un grand merci à toutes les personnes qui ont contribué à cette version. Nous n'aurions pas pu y arriver sans le soutien de la communauté.
  </p>
