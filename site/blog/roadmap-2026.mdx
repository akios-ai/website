<div class="post-content">
  <p class="lead">
    This roadmap outlines the future direction for AKIOS Open Runtime (free, GPL-3). Timelines and priorities may evolve based on community feedback.
  </p>

  <h2>Current Status – V1.0 Open Runtime (January 2026)</h2>
  <p><strong>Locked & Delivered.</strong> The minimal, trustworthy security cage.</p>
  <ul>
    <li>Kernel-hard sandbox (cgroups + seccomp)</li>
    <li>Real-time PII redaction</li>
    <li>Cryptographic Merkle tamper-evident audit</li>
    <li>Enforced cost & loop kill-switches</li>
    <li>Sequential execution only, 4 core agents</li>
    <li>Minimal CLI and polished templates</li>
  </ul>

  <h2>Future Open Releases (V1.x / V2.0)</h2>
  <p><strong>Timeline:</strong> 2026–2028 (community-driven)</p>

  <h3>V1.1 – Usability (3–6 months post-launch)</h3>
  <ul>
    <li>Full CLI suite (15–23 essential commands)</li>
    <li>Basic REST API (local-only)</li>
    <li>Improved structured logging & error messages</li>
    <li>Configuration reference generator</li>
  </ul>

  <h3>V1.2–V1.3 – Orchestration (6–12 months post-launch)</h3>
  <ul>
    <li>Parallel simple + conditional execution</li>
    <li>Loop & retry patterns</li>
    <li>Fan-out / map-reduce basics</li>
    <li>Database connectors (PostgreSQL, SQLite)</li>
    <li>GitHub agent & Webhooks receiver</li>
  </ul>

  <h3>V2.0 Horizon – Ecosystem (12–24 months post-launch)</h3>
  <ul>
    <li>Prometheus / Jaeger integration</li>
    <li>Plugin system for community agents</li>
    <li>Basic multi-tenant isolation</li>
    <li>Community marketplace for templates/agents</li>
  </ul>

  <h2>Summary</h2>
  <p>
    <strong>Open Runtime</strong> is the technical safety foundation (free).
  </p>
  <p>
    <em>The cage is ready today. The future is secure.</em>
  </p>
