# Deployment (AKIOS V1.0)

Minimal, security-first deployment: single-process runtime via pip or Docker. No sidecars, no reverse proxies, no multi-service stacks.

## Scope (what’s supported)
- **Pip package** (Linux: kernel-hard security; macOS/Windows: standard isolation).
- **Docker container** (cross-platform, policy-based security).
- Optional minimal Dockerfile and a single-service example only.

## Out of scope (V1.0)
No compose stacks, db/redis, proxies, monitoring, systemd, scaling, hardening scripts. Keep it single-container/single-binary.

> **New in V1.0.9+:** AKIOS now includes a built-in REST API via `akios serve`. See the [REST API Deployment](#rest-api-deployment) section below.

## Minimal Docker example
```dockerfile
FROM python:3.12-slim
RUN pip install akios
CMD ["akios", "run", "/app/templates/hello-workflow.yml"]
```
```bash
docker run --rm -v $(pwd)/templates:/app/templates akios:latest
```

## Security Hardening

### Run as Non-Root

The official AKIOS Docker image runs as a non-root user by default (UID 1000) since v1.0.10. If you build a custom image, always add a non-root user:

```bash
# Custom Dockerfile — create non-root user
RUN useradd -m -u 1000 akios
USER akios

# Or use --user flag at runtime
docker run --user 1000:1000 my-akios-app
```

### File Permissions

Secure your configuration:

```bash
# Restrict .env access
chmod 600 .env

# Read-only config
chmod 644 config.yaml

# Protect audit logs
chmod 700 audit/
```

### Network Isolation

No ports open by default (AKIOS doesn't listen):

```bash
# AKIOS runs as isolated process
# No need for --network host
docker run --rm my-akios-app
```

### API Key Management

Keep keys secure:

**✓ Do:**
- Use different keys per environment
- Rotate keys regularly
- Use Docker secrets or cloud key management
- Never commit `.env` to git

**× Don't:**
- Share keys between dev/prod
- Hardcode keys in Dockerfile
- Use the same key for multiple projects

## Monitoring & Health Checks

### Basic Health Check

```bash
# Check if AKIOS is healthy
akios status

# Exit code 0 = healthy, non-zero = unhealthy
```

### Structured Logging

Since v1.0.10, AKIOS uses Python's `logging` module with structured JSON output instead of plain `print()` calls. Set the log level via `config.yaml` or environment variable:

```bash
# Set log level
export AKIOS_LOG_LEVEL=DEBUG
```

All log entries include timestamp, level, module, and message fields for easy integration with log aggregation tools.

### Monitoring Audit Logs

```bash
# Watch audit events
tail -f audit/audit_events.jsonl

# Export for analysis
akios audit export --format json > audit-report.json
```

### Disk Space Monitoring

AKIOS writes to:
- `./data/` - Workflow inputs/outputs
- `./audit/` - Audit trail

```bash
# Monitor disk usage
df -h .

# Clean old runs
akios clean --old-runs 7
```

## Troubleshooting Production Issues

### Docker Image Stale

```bash
# Force pull latest
AKIOS_FORCE_PULL=1 ./akios status

# Or manually
docker pull akiosai/akios:v{{version}}
```

### Linux Kernel Requirements

For maximum security on Linux:
- Kernel >= 5.4
- seccomp-bpf support
- cgroups v2 enabled
- System packages: `libseccomp-dev` and `python3-seccomp`

```bash
# Check kernel version
uname -r

# Check seccomp
grep SECCOMP /boot/config-$(uname -r)

# Install required packages (Debian/Ubuntu)
sudo apt install libseccomp-dev python3-seccomp
```

### Configuration Validation

```bash
# Validate before deploying
akios status --verbose

# Check configuration
cat config.yaml
```

## REST API Deployment (v1.0.9+)

AKIOS includes a built-in FastAPI server for programmatic workflow execution:

```bash
# Start REST API server
akios serve --host 0.0.0.0 --port 8000
```

**Docker deployment with REST API:**
```dockerfile
FROM python:3.12-slim
RUN pip install akios
EXPOSE 8000
CMD ["akios", "serve", "--host", "0.0.0.0", "--port", "8000"]
```

```bash
docker run -d -p 8000:8000 \
  -v $(pwd)/.env:/app/.env:ro \
  -v $(pwd)/workflows:/app/workflows:ro \
  -v $(pwd)/audit:/app/audit \
  my-akios-api
```

**Health check:**
```bash
curl http://localhost:8000/api/v1/health
# {"status": "ok"}
```

The REST API inherits all AKIOS security features: sandbox, PII redaction, audit logging, and budget controls. See the [API Reference](api-reference.html) for endpoint details.

## What V1.0–V1.6 Doesn't Include

AKIOS V1.6 focuses on single-process and REST API security:

**Not included:**
- Docker Compose stacks
- Database/Redis integration (audit can migrate to SQLite/PostgreSQL via `akios audit migrate`)
- Reverse proxies
- systemd services
- Auto-scaling

**Now included (V1.0.9+):**
- Built-in REST API (`akios serve`)
- Local compliance dashboard (`akios dashboard`, v1.6.0+)
- Structured JSON logging (v1.0.10+)
- HTML security reports (`akios run --report`, v1.5.0+)

**Future versions** will add enterprise deployment features. For now: one container, one workflow (or REST API), maximum security.

## Legal & Compliance Notes

AKIOS is a security tool, not a guarantee. Users are responsible for compliance with applicable laws and regulations in their jurisdiction. The security cage reduces risk but does not eliminate it. Always validate outputs and run AKIOS in appropriate environments for your security needs. See the [Security](security.html) page for compliance frameworks supported by audit exports.

## Production Checklist

Before deploying to production:

**✓ Security**
- [ ] Run as non-root user
- [ ] `.env` file permissions set to 600
- [ ] Different API keys for prod/dev
- [ ] Audit logging enabled
- [ ] PII redaction enabled

**✓ Configuration**
- [ ] Production config tested
- [ ] Budget limits set appropriately
- [ ] Network access restricted (unless needed)
- [ ] Resource limits configured

**✓ Monitoring**
- [ ] Health check configured
- [ ] Disk space monitoring
- [ ] Audit log rotation
- [ ] Alert webhooks configured

**✓ Testing**
- [ ] Workflow tested in staging
- [ ] Error handling verified
- [ ] Mock mode tests pass
- [ ] Real API tests pass
