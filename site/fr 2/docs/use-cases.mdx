# Cas d'Usage et Exemples

Où AKIOS s'intègre dans votre infrastructure et vos flux de travail quotidiens.

## Scénarios Principaux

### Exécution d'Agents Tiers
Lorsque vous téléchargez un agent depuis une marketplace ou un dépôt communautaire, vous ne pouvez pas être sûr de ce qu'il fait. AKIOS vous permet d'exécuter ces agents en toute sécurité sur votre machine locale ou serveur de développement sans risquer vos données.

### Pipelines CI/CD
Intégrez AKIOS dans votre pipeline CI/CD pour tester les agents avant de les déployer. Vous pouvez vérifier qu'un agent respecte ses politiques déclarées (par exemple, il n'essaie pas d'accéder à Internet s'il ne le devrait pas).

### Traitement de Données Sensibles
Exécutez des tâches de traitement de données sensibles dans un environnement éphémère et isolé du réseau pour garantir qu'aucune fuite de données ne se produise.

## Exemples Concrets

Exemples spécifiques de la manière dont la cage de sécurité résout des problèmes pratiques.

### 1. Traitement Sécurisé de Documents (Secteur Public)

**Problème** : Les agences gouvernementales reçoivent des documents contenant des PII sensibles (noms, numéros de sécurité sociale).
**Solution** : Lire le document -> Rédiger les PII -> Résumer avec LLM -> Sauvegarder.

```yaml
name: "Résumé Sécurisé de Document"
description: "Rédige les PII et résume un PDF."
steps:
  - step: "lire_pdf"
    agent: "filesystem"
    action: "read"
    parameters:
      path: "/inputs/candidature.txt"

  - step: "analyser"
    agent: "llm"
    action: "complete"
    parameters:
      model: "gpt-4"
      # La rédaction des PII se fait automatiquement ici par la cage
      prompt: "Résumez cette candidature : {{lire_pdf.content}}"

  - step: "sauvegarder"
    agent: "filesystem"
    action: "write"
    parameters:
      path: "/outputs/resume.txt"
      content: "{{analyser.text}}"
```

**Valeur** : Traitement conforme au RGPD avec des pistes d'audit vérifiables.

### 2. Enrichissement via API Externe (Finance)

**Problème** : Enrichir les données clients via une API externe sans exposer les PII ni dépasser le budget.
**Solution** : Récupérer les données -> Rédiger -> Enrichir -> Coupe-Circuit de Coût.

```yaml
name: "Enrichissement KYC"
description: "Récupère les données utilisateur et analyse le risque."
steps:
  - step: "recuperer_donnees"
    agent: "http"
    action: "get"
    parameters:
      url: "https://api.external-data.com/user/{{user_id}}"

  - step: "analyse_risque"
    agent: "llm"
    action: "complete"
    parameters:
      model: "claude-3-opus"
      prompt: "Analysez le risque pour ces données utilisateur : {{recuperer_donnees.body}}"

  - step: "sauvegarder_rapport"
    agent: "filesystem"
    action: "write"
    parameters:
      path: "/reports/{{user_id}}_risque.json"
      content: "{{analyse_risque.text}}"
```

**Valeur** : Contrôle strict des coûts (limite de 1,00 $) et protection des PII pour les données financières sensibles.

### 3. Exécution d'Outils Sécurisée (DevOps)

**Problème** : Exécuter un script Python pour analyser des logs sans lui donner un accès complet au système.
**Solution** : Agent Outil avec liste blanche de commandes stricte.

```yaml
name: "Analyseur de Logs"
description: "Exécute un script python pour analyser les logs."
steps:
  - step: "lancer_script"
    agent: "tool_executor"
    action: "run"
    parameters:
      command: "python3 /scripts/analyze_logs.py --input /logs/server.log"
```

**Valeur** : Le script ne peut pas accéder au réseau ou aux fichiers en dehors des chemins autorisés.
