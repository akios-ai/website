## Platform Security Levels

| Environment | Security Level | Notes |
|-------------|----------------|-------|
| **Native Linux** | **Full (Kernel-Hard)** | Uses cgroups v2 + seccomp-bpf for maximum isolation. Recommended for high-security environments. |
| **Docker (Mac/Win/Linux)** | **Strong (Policy-Based)** | Provides strong container isolation, PII redaction, and audit trails. Does not enforce host filesystem permissions as strictly as native Linux. |

### Note on Docker Security
Docker mode provides **strong policy-based security** but cannot fully enforce host filesystem permissions (e.g., `chmod 444` bypass) or kernel-level seccomp filters on non-Linux hosts. For maximum paranoia or compliance environments, use **Native Linux**.

## What We Protect In V1.0

- **Security sandboxing**: Kernel-hard on native Linux, strong policy-based in Docker.
- **Syscall interception & resource quotas**: Preventing runaway processes.
- **Real-time PII redaction**: Protecting sensitive user data.
- **Enforced cost & loop kill-switches**: Preventing financial drain.
- **Merkle tamper-evident audit ledger**: Ensuring accountability.

**No guarantees**: No software is 100% secure. Users must secure their environment and validate outputs.
